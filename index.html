四、native 方法

Java 1.1 新增「Java本地接口」(Java Native Interface，JNI)，JNI 是一种包容极广的编程接口，允许我们从 Java 应用程序里调用 native 方法，native 方法由其它语言（C 、C++ 或汇编语言等）编写。native 方法用于实现 Java 无法处理的功能。

五、总结

反射是 native 方法，即 API，通过反射 API，可以运行时通过构造函数创建对象，以及获取和修改对象的属性、方法。

通过 native 方法操作反射对象，像镜子一样，将反应到对象上。

Java 是静态语言，先编译，后运行。正常也是运行时创建对象。

单例模式是面试中非常喜欢问的了，我们往往自认为已经完全理解了，没什么问题了。但要把它手写出来的时候，可能出现各种小错误，下面是我总结的快速准确的写出单例模式的方法。

单例模式有各种写法，什么「双重检锁法」、什么「饿汉式」、什么「饱汉式」，总是记不住、分不清。这就对了，人的记忆力是有限的，我们应该记的是最基本的单例模式怎么写。

单例模式：一个类有且只能有一个对象（实例）。单例模式的 3 个要点：

  1. 外部不能通过 new 关键字（构造函数）的方式新建实例，所以构造函数为私有：private Singleton(){}
  2. 只能通过类方法获取实例，所以获取实例的方法为公有、且为静态：public static Singleton getInstance()
  3. 实例只能有一个，那只能作为类变量的「数据」，类变量为静态 （另一种记忆：静态方法只能使用静态变量）：private static Singleton instance

Java 的几种运算符

发表于 2020-01-11 | 分类于 Java | | 阅读次数

3 种移位运算符

  • << : 左移运算符，num << 1，num 转换为二进制，丢弃最左边指定位数，1 位，整体向左移动 1 位，后面补 0。num << 1 相当于 num 乘以 2。最左位是符号位，0 为正数，1 为负数。移位后可能出现正数变负数的情况。
  • >> : 右移运算符，num >> 1，num 转换为二进制，丢弃最右边指定位数，1 位，整体向右移动 1 位，后面补符号位，如果 num 为正数，补 0。num >> 1 相当于 num 除以 2（num 为偶数时）。
    • int 类型 4 字节，移位 32 位相当于不移位，当大于 32 位时，先求余，再移位。long 类型 64 位取余。
  • >>> : 无符号右移运算符，丢弃最右边指定位数，整体右移指定位，忽略符号位，后面空位都以 0 补齐
