二、类对象

class 文件由 java 文件编译而来，class 文件包含常量池（属性）、方法表（coder，方法），方法表中包含构造函数？。

当类加载器将 class 文件加载进虚拟机元数据区（方法区，jdk1.7）时，虚拟机（在元数据中？）创建一个与之对应的类对象。

四、native 方法

Java 1.1 新增「Java本地接口」(Java Native Interface，JNI)，JNI 是一种包容极广的编程接口，允许我们从 Java 应用程序里调用 native 方法，native 方法由其它语言（C 、C++ 或汇编语言等）编写。native 方法用于实现 Java 无法处理的功能。

原理

运行 Main.class 时，将 libMain.dylib 载入虚拟机，JVM 调用 libMain.dylib 的 Java_Main_intMethod，传入参数，libMain.dylib 由系统直接运行，返回结果。

  • *env 用于将 java 类型数据与本地（此处为 C 语言）类型数据之间的转换
  • jint 还是 Java 数据类型，基本数据类型可以映射，不用显示转换

五、总结

反射是 native 方法，即 API，通过反射 API，可以运行时通过构造函数创建对象，以及获取和修改对象的属性、方法。

通过 native 方法操作反射对象，像镜子一样，将反应到对象上。

Java 是静态语言，先编译，后运行。正常也是运行时创建对象。
